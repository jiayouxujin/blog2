vSphere ESXi 命令行工具 govc 的使用

目标读者

此文仅仅针对 VMware vSphere虚拟化、 非 vSphere ESXi 用户可以洗洗睡了😥

不过我觉着 VMware 虚拟化对于中小型企业来讲还是不错滴，毕竟 VMware 他爹是 Dell 嘛😂，所以 vSphere 与 Dell 服务器结合起来才是天衣无缝嘛，毕竟 VMware 对 Dell 服务器下足功夫做了优化。

上上个月的时候，在研究如何在 vSphere 虚拟化平台上搓一套 kubernetes 集群出来，想着如何从底层的虚拟机动手，就发现了 govc 这个工具，而 govc 只是 govmoni 的一部分。

  • Kubernetes
  • Kubernetes Cloud Provider
  • Kubernetes Cluster API
  • Kubernetes kops

下面剽窃一段官方 README，其实下面大半部分也是剽窃翻译的 govc 官方使用文档 😥

govmomi

A Go library for interacting with VMware vSphere APIs (ESXi and/or vCenter).

In addition to the vSphere API client, this repository includes:

  • govc - vSphere CLI
  • vcsim - vSphere API mock framework
  • toolbox - VM guest tools framework

Compatibility

This library is built for and tested against ESXi and vCenter 6.0, 6.5 and 6.7.

It may work with versions 5.5 and 5.1, but neither are officially supported.

概括来讲 govmomi 是一个使用 Golang 实现用来与 VMware vSphere APIs 交互的工具库/客户端，这里面包含了 一个 vSphere 命令行工具 govc 。使用 govc 可以来直觉操控 ESXi 主机或者 vCenter。

下载和安装

由于是使用 Golang 写的，根据自己的系统类型在官方 GitHub 的 release 下载下来相应的二进制可执行文件并添加到环境变量即可食用。
